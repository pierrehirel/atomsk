Option : rebox

Syntaxe

-rebox

Description

Cette option force le programme à re-calculer les vecteurs de boîte.

ATTENTION : cette option tente de définir une boîte parallélépipédique englobant tous les atomes (elle ne sera donc PAS correcte pour des systèmes tricliniques), et est très imprécise. Ne faites pas confiance à cette option : vérifiez les vecteurs de boîte !

Si vous connaissez les vecteurs de boîte de façon précise, il est recommandé d'utiliser l'option -properties au lieu de la présente option.

Défaut

Par défaut atomsk utilise toujours les vecteurs de boîte lus depuis le fichier d'entrée. Cependant si aucun vecteur de boîte n'est trouvé dans le fichier d'entrée, cette option est automatiquement invoquée juste après la lecture du fichier, et avant l'application de toute autre option. C'est le cas par exemple du format XYZ par défaut, qui ne contient pas de vecteur de boîte.

Exemples

  • atomsk system.gin -rebox xsf

    Ceci lira le fichier system.gin et re-calculera les vecteurs de la boîte. Le résultat final sera écrit dans system.xsf.

  • atomsk exemple.cfg -cut above 12 z -rebox xsf xyz

    Ceci va couper le système exemple.cfg au-dessus de Z=12 Å, puis re-calculera les vecteurs de boîte. Le résultat sera écrit dans exemple.xsf et exemple.xyz.
