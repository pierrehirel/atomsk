Outils companion

Le dossier /tools/ fourni avec le programme contient des scripts bash qui peuvent être utilisés pour opérer des transformations simples sur des fichiers. Lors de l'installation de atomsk (soit avec le script "install.sh", soit avec la commande "make install"), ces scripts sont rendus exécutables et placés dans le dossier /usr/local/bin. Exécuter l'un de ces scripts sans argument affichera son utilisation.

Comme le reste du code, ces scripts bash sont fournis tels quels et sans aucune garantie. Il est recommandé de les éditer pour vérifier ce qu'ils font et/ou les adapter à vos besoins.

cfg_setA.sh

Ce script donne au facteur d'échelle A la <valeur> donnée dans un ou plusieurs fichiers Atomeye (*.cfg). Ce script peut être utile si Atomeye refuse d'afficher un système et affiche le message d'erreur "ATOM_COORDINATION_MAX = 24 exceeded". Par exemple, pour donner la valeur A=1.3 dans le fichier "monfichier.cfg" entrez :

cfg_setA.sh 1.3 monfichier.cfg

Si plusieurs noms de fichiers sont donnés, ils seront tous convertis. Attention, ce script ré-écrit et remplace le fichier d'entrée.

dat_mulvec.sh

Ce script bash lit un fichier de données (*.dat) contenant au moins six colonnes de nombres réels, en supposant qu'il s'agit des positions d'atomes et des composantes d'un vecteur (X Y Z Vx Vy Vz), et multiplis les composantes des vecteurs par le <facteur> donné. Pour convertir le fichier "monfichier.dat", entrez :

dat_mulvec.sh <facteur> monfichier.dat

Plusieurs noms de fichiers peuvent être donnés. Attention, ce script ré-écrit et remplace le fichier d'entrée.

dat_rm0.sh

Ce script bash lit un fichier de données (*.dat) contenant au moins six colonnes de nombres réels, en supposant qu'il s'agit des positions d'atomes et des composantes d'un vecteur (X Y Z Vx Vy Vz), et supprime les lignes pour lesquelles les trois dernières valeurs sont zéro. Pour convertir le fichier "monfichier.dat", entrez :

dat_rm0.sh monfichier.dat

Plusieurs noms de fichiers peuvent être donnés. Attention, ce script ré-écrit et remplace le fichier d'entrée.

lmp_atom2charge.sh

Ce script bash convertit un ou plusieurs fichiers de données de LAMMPS (*.lmp) au format "ID type x y z" (adapté pour le style "atom_style atom") vers le format "ID type charge x y z" (adapté pour le style "atom_style charge"). Pour convertir le fichier "monfichier.lmp", entrez :

lmp_atom2charge.sh monfichier.lmp

Plusieurs noms de fichiers peuvent être donnés. Attention, ce script ré-écrit et remplace le fichier d'entrée. Dans la pratique ce script copie les deux premières colonnes de données, ajoute des zéros dans la troisième colonne, et copie les trois dernières colonnes de données.

lmp_charge2atom.sh

Ce script bash fait l'inverse du script précédent, c-à-d. convertit un ou plusieurs fichiers de données de LAMMPS (*.lmp) au format "ID type charge x y z" (adapté pour le style "atom_style charge") vers le format "ID type x y z" (adapté pour le style "atom_style atom"). Pour convertir le fichier "monfichier.lmp", entrez :

lmp_charge2atom.sh monfichier.lmp

Plusieurs noms de fichiers peuvent être donnés. Attention, ce script ré-écrit et remplace le fichier d'entrée. Dans la pratique ce script copie les deux premières et les trois dernières colonnes de données.

lmp_ortho2tri.sh

Ce script bash transforme un fichier de données de LAMMPS (*.lmp) de sorte à rendre le système triclinique. Après la ligne se terminant par "zhi", la nouvelle ligne suivante est ajoutée : "0.0000000 0.00000000 0.00000000 xy xz yz". Si une ligne contenant "xy xz yz" existe déjà, alors le script ne fait rien. Comme pour les scripts précédents, pour convertir le fichier "monfichier.lmp" entrez :

lmp_ortho2tri.sh monfichier.lmp

Plusieurs noms de fichiers peuvent être donnés. Attention, ce script ré-écrit et remplace le fichier d'entrée.

qepw_bohr.sh

Ce script bash modifie les lignes "CELL PARAMETERS" et "ATOMIC_POSITIONS" pour leur ajouter le mot "bohr", dans un ou plusieurs fichiers d'entrée pour Quantum Espresso pwscf (*.pw). En effet lorsqu'il écrit un fichier au format PW atomsk ne connaît pas les unités des coordonnées et suppose toujours qu'il s'agit d'angstroms. Ce script permet de corriger cela si les coordonnées sont en réalité en Bohrs. Pour convertir le fichier "monfichier.pw", entrez :

qepw_bohr.sh monfichier.pw

Plusieurs noms de fichiers peuvent être donnés. Attention, ce script ré-écrit et remplace le fichier d'entrée.
