Format: MOLDY

Name: configuration for MOLDY

Extension: unspecified (atomsk uses .mol)

Specification: MOLDY Wiki

Visualization programs: N/A

Restrictions

atomsk can read MOLDY configuration files (e.g. "system.in" or "system.out"). As stated in the specification, the second line contains three integers that correspond to the number of replicas along the X, Y and Z directions. atomsk will read these numbers and actually perform the replication (i.e. these integers have the same effect as the option -expand).

When writing to the MOLDY format, atomsk will use ".mol" for the file extension. The atom masses that are written are those of the most common isotopes as provided by the National Institute of Standards and Technology (NIST).

Examples

  • atomsk my_system.cfg mol

    This will read the file my_system.cfg, and convert it to my_system.mol.
